swf 2 lottie

        
Swf 2 lottie is a vibecoded tool to convert Adobe Flash / Animate SWF files to Lottie JSON format.

        
I made it 'cause I needed some short animations for my other project and God knows I hate to grind SWFs through After Effects! So, for now it seems it fully meets my needs.

        
For full details and instructions on how to run in locally, please see README.md in the GitHub repository for more details and examples: https://github.com/takopus/swf2lottie

        
Disclaimer: This is a hobby project and not affiliated with Adobe or Lottie. It was vibecoded to life, I never dug deep into the code and give no guarantees it won't install some backdoor for future AI masters, if you clone and use it locally!


        
Supported features:
          
    
            
  • Vector shapes: fills and outlines, including (most) line styles and linear and radial gradients;
    • 
            
  • Basic transformation: rotation, scaling, skewing;
    • 
            
  • Movie clips and nested symbols - your usual scene hierarchy;
    • 
            
  • Classic and motion tweens;
    • 
            
  • Shape tweens;
    • 
            
  • Bitmaps, except for bitmap outlines and some other rare cases.
    •             
          

        


        
Unsupported features:
          
    
            
  • Text fields. Convert them to shapes by breaking apart;
    • 
            
  • Sound and video;
    • 
            
  • ActionScript and interactivity;
    • 
            
  • 3D transforms;
    • 
            
  • Filters and blend modes;
    • 
            
  • Everything else outside the supported features.
    • 
          

        


        
I seriously doubt it will work with no bugs. If you encounter any, please try to refine it to a minimal test case and report an issue in the GitHub repository with FLA / SWF files. I will try to feed it into my trusted OpenAI Codex, if I still have access to it by that time. Thanks!

        
I also thought about making Adobe Animate IDE extension for this tool, but it seems JSFL still more dead than alive, so you'll have to stick with web service, at least for now. If anyone want to try their hand at it, feel free to do so - Codex says it won't be that hard on this project's side, but may require a lot of luck and nerves on JSFL end.

        
Have fun making web animations the way they meant to be made, in an IDE designed specifically for that purpose, and not in the, you know, After Effects!
